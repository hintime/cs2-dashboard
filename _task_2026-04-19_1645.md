# 任务摘要：CSQAQ Skill 集成 + 智能切换机制

## 目标
将 `csqaq-market-lookup` skill 接入 update.py，并实现两种调用方式互补。

## 完成内容

### 1. 新增 Skill 调用函数
- `fetch_csqaq_via_skill()` — 通过 skill CLI 调用 CSQAQ API
- 使用 `--body-file` 传递 JSON，避免 PowerShell 参数编码问题
- 使用 GBK 编码解析 skill 输出

### 2. 保留直接 HTTP 调用
- `_fetch_csqaq_direct()` — 原有的直接 HTTP 调用逻辑

### 3. 实现智能切换机制
```python
fetch_csqaq_alerts(use_skill='auto')  # 默认：自动切换
```

**切换逻辑：**
1. 优先尝试直接 HTTP 调用（更快）
2. 如果失败或返回空，自动切换到 skill 调用
3. 可强制指定调用方式

### 4. 修复的问题
- Skill 输出中 `[CALL]` 行被误识别为 JSON（都以 `[` 开头）
- 修复：跳过 `[CALL]` 和 `[STATUS]` 开头的行

## 测试结果
- ✅ 直接调用正常：71 异动，28-30 推荐
- ✅ Skill 调用正常：返回数据正确
- ✅ 智能切换生效：自动选择最佳方式
