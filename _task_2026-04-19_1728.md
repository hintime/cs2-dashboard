# 任务总结：增加「求购异动」「在售异动」标签

**时间**: 2026-04-19 17:28
**状态**: ✅ 已完成

---

## 目标
在饰品异动板块增加「求购异动」和「在售异动」两个标签，方便用户监测求购量和在售量的变化。

---

## 实现方案

### 方案 A（最初想法）
依赖 CSQAQ API 的排序参数（如 `sell_up_1d`, `buy_up_1d`）获取不同类型的异动数据。

**测试结果**: API 返回的数据相同，sort 参数似乎被忽略。

### 方案 B（最终采用）
前端按字段排序：
- **涨幅榜**: 筛选 `rate_1 > 0`，按涨跌幅降序
- **跌幅榜**: 筛选 `rate_1 < 0`，按涨跌幅升序
- **求购异动**: 按 `buff_buy`（求购量）降序
- **在售异动**: 按 `buff_sell`（在售量）降序
